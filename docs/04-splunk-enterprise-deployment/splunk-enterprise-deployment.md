\# 04 - Splunk Enterprise Deployment



\## 1. Objective



To establish centralized security monitoring within the enterprise environment, Splunk Enterprise was deployed as the Security Information and Event Management (SIEM) platform. Windows event data generated by domain-joined endpoints is collected using the Splunk Universal Forwarder, providing a centralized platform for log collection, searching, and security investigations.



\---



\## 2. Technologies Used



| Component | Technology |

|---|---|

| SIEM Platform | Splunk Enterprise 10.4.1 |

| Operating System | Ubuntu Server 22.04 LTS |

| Log Collection Agent | Splunk Universal Forwarder |

| Monitored Endpoint | Windows 11 (WS01) |

| Transport Protocol | TCP |

| Receiving Port | 9997 |



\---



\## 3. Splunk Enterprise Deployment



Splunk Enterprise was installed on a dedicated Ubuntu Server virtual machine to provide centralized log management and security monitoring.



The web interface was configured to allow administrative access over TCP port \*\*8000\*\*, while a receiving port was configured on TCP port \*\*9997\*\* to accept incoming data from Splunk Universal Forwarders.



This dedicated server serves as the centralized repository for security events generated throughout the enterprise environment.



\---



\## 4. Universal Forwarder Deployment



The Splunk Universal Forwarder was installed on \*\*WS01\*\* to collect Windows event data and securely forward it to the Splunk Enterprise server.



The forwarder was configured using the following settings:



| Setting | Value |

|---|---|

| Destination Server | 192.168.126.144 |

| Receiving Port | 9997 |



After configuration, the Universal Forwarder service was started and verified to ensure successful communication with the Splunk Enterprise server.



\---



\## 5. Log Collection Configuration



The Universal Forwarder was configured to collect the following Windows Event Logs:



\- Windows Security

\- Windows System

\- Windows Application

\- Microsoft-Windows-PowerShell Operational

\- Microsoft-Windows-Sysmon Operational



Collecting these logs provides visibility into authentication activity, operating system events, PowerShell execution, and endpoint telemetry required for security monitoring and investigations.



\---



\## 6. Verification



The deployment was verified by confirming:



\- Splunk Enterprise was running successfully on the Ubuntu server.

\- TCP port \*\*9997\*\* was configured to receive forwarded events.

\- The Universal Forwarder service was running on WS01.

\- The Universal Forwarder established an active connection to the Splunk Enterprise server.

\- Windows Event Logs were successfully indexed by Splunk.

\- WS01 appeared as the event source within the Search \& Reporting application.



These verification steps confirmed successful communication between the monitored endpoint and the centralized SIEM platform.



\---



\## 7. Outcome



A centralized logging solution was successfully deployed within the enterprise environment.



Security events generated by WS01 are now automatically forwarded to Splunk Enterprise, where they can be searched, analyzed, and correlated during future security investigations.



This deployment establishes the foundation for centralized monitoring, detection engineering, and incident response throughout the remainder of the project.



\---



\## 8. Next Phase



The next phase focuses on deploying the Splunk Universal Forwarder to the remaining Windows systems within the environment to expand centralized log collection and prepare the infrastructure for detection engineering.
